# Decisions log

Deviations and substitutions from the build-spec, per the rule: substitute the
nearest feasible option and log it — never silently lower the bar.

| # | Date | Decision | Why | Spec impact |
|---|------|----------|-----|-------------|
| 1 | 2026-07-03 | **Electron instead of Tauri 2** | Tauri needs Rust + MSVC build tools (~8–10 GB, not installed, ~28 GB disk free). Electron runs on the existing Node 24 toolchain with prebuilt binaries and matches the OpenWhispr blueprint cited by the spec. Spec explicitly allows this fallback. | None — spec-sanctioned fallback. |
| 2 | 2026-07-03 | **whisper.cpp prebuilt sidecar process** (v1.9.1 cublas-12.4 build) instead of faster-whisper or node bindings | Python is not installed (faster-whisper impossible); whisper.cpp node bindings require MSVC to compile. The prebuilt binary needs no toolchain and bundles CUDA DLLs for the RTX 3060. | None — spec lists whisper.cpp as a first-choice engine. |
| 3 | 2026-07-03 | **qwen3:4b-instruct** as the default cleanup model (spec-era examples suggested 3–8B class) | 2026 rankings place qwen3-4B at the top of the small-instruct class (~2.5 GB, fits 6 GB VRAM next to whisper). The hybrid-thinking `qwen3:4b` variant burned 2,000+ chain-of-thought tokens per request (~50 s); the non-thinking `-instruct` variant answers in ~0.3–1.3 s and passes the full fixture suite with two deterministic post-processing rules (list-marker normalization, messaging trailing-period). | Positive — full contract passes at ~40× lower latency. |
| 4 | 2026-07-03 | **Hold-to-talk via uiohook-napi; toggle via globalShortcut as automatic fallback** | Electron's globalShortcut API has no keyup events, so true push-to-talk needs a low-level hook. If the native hook fails to load, the app degrades to toggle mode rather than breaking. | None — both modes are spec features. |
| 5 | 2026-07-03 | **SQLite via better-sqlite3, with a JSON-file fallback store** compiled in | better-sqlite3 Electron prebuilds have gaps on the newest ABIs; Electron pinned to 38.x where prebuilds exist. If the prebuild still fails to load at runtime, the app falls back to a JSON store with the same interface instead of crashing. | Minor — fallback is not SQLite, but data model and privacy are identical. |
| 6 | 2026-07-03 | **Native Android/iOS apps deferred; Phase 6 ships a LAN-bridge phone page** | iOS cannot be built on Windows at all; the Android SDK toolchain exceeds this session's scope/disk. Instead: the desktop app hosts an opt-in, token-protected page on the user's Wi-Fi — the phone records and the PC's local models do STT+cleanup, so mobile stays private and free (no cloud at all). Audio crosses only the user's own network. | Logged Tier-2 follow-up per spec's "nearest feasible option" rule; arguably stronger privacy than the spec's cloud-fallback mobile path. |
| 7 | 2026-07-03 | **qwen3 chain-of-thought suppressed via /no_think + output-format contract + sanitizer rescue** | qwen3:4b leaks reasoning into content on some inputs (with \boxed{} answers). Fixed at three layers: soft switch in the system prompt, an ABSOLUTE output-format rule, and a sanitizer that extracts boxed/post-</think> text. | None — determinism and cleanliness preserved. |
